# AceBook 小程序版本归档索引

> 用途：按版本回查的总入口。每发一个体验版，必须在此登记，并打同名 Git tag。
> 仓库：github.com/cqujgcc-bit/acebook-miniapp（权威源码 = 云电脑 /home/ubuntu/miniapp）

## 如何按版本回查

```bash
# 1) 列出所有已归档版本
git tag

# 2) 查看某版本改了什么（说明文档）
cat RELEASES/v1.5.11.md

# 3) 取出某版本的完整代码（只读查看，不影响当前分支）
git checkout v1.5.10        # 切到该版本
git checkout master         # 回到最新

# 4) 只导出某版本到一个目录（不切分支）
git archive v1.5.10 --format=tar --prefix=acebook-v1.5.10/ | tar -x -C /tmp/
```

## 归档规范（每个新版本必做）

1. 代码改完、体验版上传成功后，在本仓库 commit（message 含版本号）。
2. 打 tag 并推送：`git tag -a v1.x.x -m "说明"` → `git push origin v1.x.x`。
3. 在 `RELEASES/` 新建 `v1.x.x.md`（用模板 `RELEASES/_TEMPLATE.md`）。
4. 更新本 README 的"版本登记表"。
5. 在云电脑 AGENTS.md 追加变更记录。
