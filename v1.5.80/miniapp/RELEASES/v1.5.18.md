## 一、核心需求（用户第⑥项）

发起人可在球局结束后，按**实际到场人数**重新均摊场地费，并通过微信完成**真实多退少补**：

- 到场者**多付**的部分 → 原路退回微信；
- 到场者**少付**的部分 → 生成待补缴账单（不强制划扣，提醒补缴）；
- **爽约**（缺席）者 → 已付费用不退还，并扣除信用分；
- 最终场地费实收结算到发起人微信零钱。

并支持发起人**编辑球局**（标题/时间/地点/名额/费用）与**动态调整人数**。

## 二、关键产品规则：发起人是否分摊场地费

> **发起人参加球局，则与球友一同分摊场地费；未参加则不分摊。**（用户明确强调）

- 结算确认页新增「我本人是否到场打球」勾选。
- **发起人到场**：分摊总人数 = 付费到场人数 + 1（含发起人），人均 = 场地费 ÷ 分摊总人数；发起人自付其一份，该份额自动从结算金额中扣除（发起人不把自己那份收走）。
- **发起人未到场**：分摊总人数 = 付费到场人数，发起人不分摊，实收全额结算给发起人（兼容原逻辑）。

### 结算算法（分单位计算，避免浮点精度问题）

```
divisor   = paidAttendedCount + (organizerAttended ? 1 : 0)
perFen    = round(courtTotalFee*100 / divisor)
对每位付费到场者：diff = paidFen - perFen；diff>0 退 diff，diff<0 补 -diff
organizerShareFen = organizerAttended ? perFen : 0
结算给发起人 = totalPaidFen - refundTotalFen - organizerShareFen
```
