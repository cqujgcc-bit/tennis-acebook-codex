## 一、本次目标

落实第③项（超级管理员权限扩展）与第④项（用户反馈查看与回复闭环），合并为统一的「管理后台」页面体系。仅 `role=admin` 的账号在「我的」页面可见入口，普通用户完全不可见、不可达。

## 二、新增功能

### 1. 管理后台首页（pages/admin）
- 顶部数据看板：教练数、学员数、累计球局、本周球局/参与人次（来自 `admin.stats`）。
- 功能导航卡片：人员管理、教练审核、用户反馈、全局公告、预订平台排布。

### 2. 人员管理（pages/admin-users）
- 用户列表展示昵称、ID、**授权手机号**、信用分、角色、封禁状态。
- 搜索支持昵称 / 手机号 / openId（后端 `listUsers` 搜索条件已补充 phone）。
- **信用分调整**：快捷档（±5/±10/恢复满分）+ 自定义增减，必填原因，写入信用分日志并通知用户（`admin.adjustCredit` → 复用 `addCreditLog`，钳制 0–100）。
- 警告（`admin.warnUser`）、封禁/解封（`admin.banUser`/`unbanUser`）。
- 分页加载（触底翻页）。

### 3. 教练审核（pages/admin-coaches）
- 拉取待审核教练（`admin.pendingCoaches`），展示资质、专长、电话、简介、证书图片（可预览）。
- 通过（`admin.approveCoach`）/ 驳回并填原因（`admin.rejectCoach`），结果通知教练。

### 4. 全局公告（pages/admin-broadcast）
- 标题 + 正文 + 发送范围（全体 / 仅普通用户 / 仅教练）。
- 以「系统通知」批量推送至用户消息中心（`admin.broadcast`），返回实际送达人数。

### 5. 预订平台排布（pages/admin-venues + pages/admin-venue-edit）
- 「找场地」页的官方预订小程序清单迁移到后端 `booking_apps` 表，管理员拥有决策权：
  - 上移/下移调整顺序后「保存顺序」（`admin.reorderBookingApps`，按位置重算 sortWeight）。
  - 上架/下架（`admin.toggleBookingApp`）。
  - 新增 / 编辑 / 删除（`admin.upsertBookingApp` / `admin.deleteBookingApp`）。
- 「找场地」页（pages/venues）改为优先读取后端 `venue.bookingApps`（公开接口，按权重降序、仅返回上架项），失败时回退本地 `utils/venues.js` 兜底，避免白屏。

### 6. 用户反馈回复闭环（pages/admin-feedback）
- 查看全部反馈，支持「全部 / 待处理 / 已回复」筛选（`feedback.adminList`）。
- 展示用户、分类、状态、内容、联系方式、时间与历史回复。
- 一键回复（`feedback.reply`）：写入 adminReply + 状态置为 replied，并自动给用户发系统通知；用户在「意见反馈」中可见官方回复。
